# Vault Index

Updated: on init — update this file whenever vault structure changes.

---

## Folder Map

| Path | English aliases | Notes |
|---|---|---|
| `inbox/` | inbox, capture, unsorted, new notes | All new captures land here |
| `meta/` | meta, agent files, index, contract | vault-index.md, topic-index.md, va-contract.md, reports/ |
| `meta/reports/` | routine outputs, monthly reports, quarterly reports | Auto-generated by monthly_audit (YYYY-MM.md) and quarterly_review (YYYY-Qn.md) |
| `notes/` | notes, evergreen, permanent notes | Evergreen notes |
| `projects/` | projects, active projects | Active project notes |
| `areas/` | areas, responsibilities, ongoing | Ongoing areas of responsibility |
| `areas/people/` | people, contacts, relationships | People notes |
| `resources/` | references, resources, reading, links | Reference material |
| `templates/` | templates | Note templates |
| `mocs/` | mocs, maps of content, index notes | Maps of content — add your own role MOCs here |
| `archive/` | archive, archived, old | Archived notes |
| `assets/` | assets, images, attachments | Vault-wide attachments — single canonical location for all files. No `files/` sibling dirs. |

---

## How to Regenerate

When vault structure changes (new folders, renamed paths):
1. Run on VM: `find ~/repos/ctb-vault -maxdepth 3 -type d`
2. Update the table above — add new rows, remove deleted folders, update aliases as needed
3. Save and commit: `agent(vault): update vault-index`
